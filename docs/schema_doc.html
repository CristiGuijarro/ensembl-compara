EnsEMBL Compara DB Schema Documentation

This document refers to the schema of the EnsEMBL Compara version 22.1. This is the revision 0.1 of this document.

Introduction

EnsEMBL Compara DB is divided into two parts:
  • Genomic alignments

    This part is dedicated to DNA-DNA alignment. It includes alignment made with different alignment tools. It also includes a list of syntenic regions.

  • Protein alignments

    This part contains several types of alignments.

    The first one corresponds to a list of protein orthologues. By now, for each gene, only the longest peptide is considered for homology search. Only pairwise comparisons are taken into account. In principle, only Best Reciprocal Hits (BRH) are considered. Nevetheless extra orthologous relationships are found taking into account the synteny of neighbour genes (RHS, Reciprocal Hit based on Synteny around BRH). Additionally, in Human vs. Chimp relationships may be found Derived from Whole Genome Alignment (DWGA).

    Another part is dedicated to protein clusters. In this case, Metazoan entries of the SwissProt/SPTrEMBL database are also taken into account in order to annotate the clusters.

    A third part is planed for domain alignments but is not used by now.

genomic_align_block

This table contains all the genomic alignments. It is the main table for this part of EnsEMBL Compara. It defines the aligned genomic blocks using external references to the dnafrag table. The software used to align the genomic blocks is refered as an external key to the method_link table.

The original alignment is not stored but it can be retrieved using the cigar_line field and the original sequences. The cigar line defines the sequence of matches, insertions and deletions in the alignment. For example, this cigar line 2MI3M2D2M will mean that the alignment contains 2 matches, 1 insertion (number 1 is omitted in order to save some space), 3 matches, 2 deletions and 2 matches. If the original sequences are:

  • Consensus: AACGTCTT
  • Query:     AAGTCAATT

The alignment will be:

cigar line: 2MI3M2D2M
dnafrag M M I M M M D D M M
Consensus A A C G T C - - T T
Query A A - G T C A A T T

genomic_align_block table description
Field Type Null Key Default Extra Description
consensus_dnafrag_id int(10) MUL 0 external reference to dnafrag.dnafrag_id
consensus_start int(10) 0 consensus DNA coordinates. consensus strand is 1 by default.
consensus_end int(10) 0
query_dnafrag_id int(10) MUL 0 external reference to dnafrag.dnafrag_id
query_start int(10) 0 query DNA coordinates
query_end int(10) 0
query_strand tinyint(4) 0
method_link_id int(10) 0 external reference to method_link.method_link_id
score double YES NULL score returned by the homology search program
perc_id int(10) YES NULL percentaje of identity
cigar_line mediumtext YES NULL internal description of the alignment
group_id int(10) 0 used to group alignments
level_id int(10) 0 level of orhologous layer. 1 corresponds to the first layer of orthologous sequences found, 2 and over are addiotional layers. Use for building the syntenies (based on level_id = 1 only)
strands_reversed tinyint(1) 0 1 when the alignment should be reverse complement to restore the original one (only used with TRANSLATED_BLAT results)

Consensus_dnafrag_id, consensus_start and consensus_end fields define the consensus region aligned with the query ones. Consensus regions are in the forward strand by default. Query regions are defined with query_dnafrag_id, query_start, query_end and query_strand. The query strand could be 1 if the query strand matches the consensus one or -1 otherwise. The strands_reversed is 0 by default. It is equal to 1 when the alignment should be reverse complement to restore the original one.

DNA strands
query_stand strands_reversed consensus DNA strand query DNA strand
1 0 forward forward
1 1 reverse reverse
-1 0 forward reverse
-1 1 reverse foward

seq_end int(10) unsigned 0 last nucleotide from this dnafrag which is in synteny

E.g. the rows

+-------------------+------------+-----------+----------+
| synteny_region_id | dnafrag_id | seq_start | seq_end  |
+-------------------+------------+-----------+----------+
|               989 |          5 |   8196083 |  9099311 |
|               989 |         31 |  66462540 | 67445576 |
+-------------------+------------+-----------+----------+

correspond to both genomic regions of the syntenic region 989. In this case, the first genomic region corresponds to the sequence from 8196083 to 9099311 of the chromosome X of the human genome (dnafrag_id = 5 for this chromosome) and the second one corresponds to the sequence from 66462540 to 67445576 of the chromosome X of the mouse genome (dnafrag_id = 31 for this chromosome). Using the synteny_region table, we know that both syntenic regions are in the same orientation.

Contains all the genomic homologies found. There are two homology_member entries for each homology entry for now, but both the schema and the API can handle more than just pairwise relations.

homology_member

The original alignment is not stored but it can be retrieved using the cigar_line field and the original sequences. The cigar line defines the sequence of matches or mismatches and deletions in the alignment.

  • First peptide sequence:  SERCQVVVISIGPISVLSMILDFY
  • Second peptide sequence: SDRCQVLVISILSMIGLDFY
  • First corresponding cigar line:  20MD4M
  • Second corresponding cigar line: 11M5D9M

The alignment will be:

Second aligned peptide S D R C Q V L V I S I - - - - - L S M I G L D F Y
